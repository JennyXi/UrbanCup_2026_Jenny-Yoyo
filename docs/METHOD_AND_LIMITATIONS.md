# 方法、识别边界与限制

## 研究问题

本项目研究在极端高温和强降雨下，不同数字接入能力的人群如何选择交通方式，以及网约车优惠券、数字协助和派单优先规则如何改变需求、有限车辆竞争、户外暴露和必要活动完成。

## 合成城市与 Agent

九个功能区表达“就业中心—副中心—内城—外围居住—产业新城—远郊弱势区”的空间结构，不复制上海真实行政边界。Agent 按三年龄组生成，并包含就业、医疗需求、智能手机、独立数字接入和家庭协助等属性。

活动和 OD 使用固定 seed 的稳定哈希与显式概率生成。工作和医疗地点按 Agent 固定；非必要活动允许受天气取消；连续 legs 必须满足活动时间、旅行时间、营业时间和年龄返家上限。

## 交通与派单机制

Agent 比较完整门到门时间、候车、接驳、换乘、费用、预计网约车等待、天气暴露偏好、可靠性缓冲和迟到惩罚。可用方式包括步行、公交、地铁、公交—地铁组合和网约车。

网约车采用逐车空间—时间守恒：成功订单使车辆从起点池进入 busy，行程结束后在目的地区域释放。失败后最多进行一次 fallback，并使用已经消耗的时间、当前位置和剩余预算重新选择。

公交和地铁当前保证上车；道路公交和网约车共享天气及道路拥堵，地铁车内运行不受道路减速影响，但步行和公交接驳仍受天气影响。

## 政策设计

- C0：无券；
- C1：全年龄有限公共券；
- C2：老年定向有限券；
- C3：公共券与老年保留券混合。

券在首次网约车请求时绑定，成功后核销，失败后按配置失效。政策比较在 seed 内共享人口、活动、OD、天气和基础随机量。

## 统计口径

- 系统结果首先按 seed 输出，不先聚合后伪造样本量；
- 政策效应使用同一 seed 的政策值减 C0；
- 报告配对均值和 95% Student t 区间；
- 区间跨越 0 时，只描述样本方向，不称为稳定效应；
- 群体表明确实际有效 seed 数，不用人为 0 或 100% 填补不存在的群体—活动组合。

当前 10 seeds 主要用于机制验证。下一阶段建议预先冻结 30–100 seeds、主要结局和停止规则，再运行稳健性实验，避免根据单个 seed 调参。

## 公平性指标

当前正式输出包括：

- 必要活动完成率和交通未满足；
- 平均及高分位等待时间的基础记录；
- 年龄 × 数字接入 × 家庭协助群体结果；
- 优惠券触达、参与、获得、核销与诱发请求；
- 脆弱组相对年轻组的必要活动完成率差距；
- 户外暴露和年龄加权热风险。

正式政策评价还应增加预注册的 P90/P95 等待、区域机会差距、最大最小公平和预算约束下的受益分配。指标增加必须在看正式结果之前冻结。

## 等预算问题

C1–C3 当前都配置为相同发券数量，但核销与补贴支出不同，所以不是严格等财政预算。结果卡提供元/核销和元/诱发请求，仅用于描述机制效率。

正式等预算实验应先确定共同预算，再让不同政策在相同实际补贴上限下竞争；政策排序应同时报告总福利、脆弱组收益和其他群体外部性。

## 参数证据

人口结构、数字接入和天气—交通扰动尽可能链接公开证据。缺乏统一观测支持的速度、等待、车辆供给、偏好和容量参数显式标记为 `model_assumption`。

现实外推前至少需要：

1. 上海观测 mode share、OD、出发时间与网约车订单率的联合校准；
2. 留出数据或独立年份的外部验证；
3. 车辆供给、天气严重度、优惠额度和行为参数的全局敏感性；
4. 合成 Agent 权重到居民/出行/PCU 的正式映射。

## 已知限制

- 九区空间与网络是合成结构，不是 GIS 精确复刻；
- 200 Agents 适合机制测试，不是代表性人口样本；
- 公交与地铁没有容量、拥挤拒载和站内排队；
- 道路反馈不是完整动态用户均衡；
- 没有司机行为、拼车、动态加价、长期学习与居住迁移；
- 交叉群体样本较小，部分公平性区间很宽；
- 当前政策不是严格等预算；
- AgentSociety 大规模端到端结果尚未纳入正式证据。

## 允许与不允许的表述

允许：

- “在当前合成参数和 10 seeds 中，C1/C3 稳定增加网约车请求。”
- “有限车辆竞争产生局部派单失败，但 fallback 吸收了多数冲击。”
- “老年定向券的触达与实际核销之间存在明显断层。”

不允许：

- “该政策将使上海必要出行提高 X%。”
- “36 辆车是上海最优供给比例。”
- “发券 40 张的政策具有相同财政成本。”
- “组成均值道路速度上升等于全城拥堵改善。”
