# 九区50-Agent优惠券竞争实验

本实验在供给阈值实验选出的12辆工作日日初网约车基线上，对C0–C3优惠券政策进行配对比较。它是机制实验，不是上海真实优惠券领取率、运力或福利预测。

## 固定条件

- 50 Agents；九个功能区。
- walking、bus、metro、ride_hailing。
- W0正常天气和W2普通强降雨工作日。
- seeds 47–56，共10个 seeds。
- 每个seed跨政策保持Agent、活动、OD、天气、12辆车、方式选择扰动和基础派单优先值一致。
- 网约车车辆严格遵守空间—时间守恒；公交和网约车共享道路。
- 每日券池10张，每张八折，每人最多获得和核销1张。

## 政策

- `C0_no_coupon`：无券。
- `C1_public_limited`：全年龄有限公共抢券。18–39、40–59、60+参与概率分别为0.80、0.65、0.45；非数字且无协助老人不能参与。
- `C2_elder_limited`：仅向可触达60+老人发放；数字老人直接领取，非数字但有家庭协助者由家人代收。
- `C3_mixed`：70%公共池、30%老年保留池；老年保留池增加40%的有限社区/电话覆盖渠道。

券只在Agent当天第一次网约车请求时绑定。成功派单才核销；首次请求失败后券失效。优惠只改变网约车费用效用，不直接指定交通方式，也不修改数字接入身份。

## 10-seed主要结果

### W2强降雨

| 指标（每seed均值） | C0 | C1公共券 | C2老年券 | C3混合券 |
|---|---:|---:|---:|---:|
| 网约车请求 | 13.0 | 14.7 | 13.3 | 14.1 |
| 优惠券诱发请求 | 0.0 | 1.7 | 0.3 | 1.1 |
| 成功订单 | 11.6 | 13.2 | 11.6 | 12.6 |
| 失败订单/fallback | 1.4 | 1.5 | 1.7 | 1.5 |
| 每请求平均等待（分钟） | 11.97 | 12.02 | 12.19 | 11.94 |
| 券核销数 | 0.0 | 3.6 | 0.1 | 2.4 |
| 平均补贴（元） | 0.00 | 26.53 | 0.41 | 16.31 |
| 必要活动完成率 | 97.05% | 96.78% | 97.05% | 96.78% |
| 平均总通行时间（分钟） | 64.22 | 63.76 | 64.28 | 63.94 |

C1和C3产生了额外网约车请求，也提高了成功订单和道路车辆量；等待及失败变化较小，fallback仍吸收全部首次失败，没有出现最终`transport_unmet`。在本50-Agent规模下，公共券造成了可见但较弱的资源竞争，尚未形成系统性挤出。

C2平均发出约9.9张老年券，但W2平均只核销0.1张。新增的0.3次老年网约车请求主要发生在非数字但有家庭协助组，并在有限样本中未转化为更多成功订单。这说明“发到券”不等于“产生有效可达性改善”：老年人的方式偏好、出行需求、起点车辆可用性仍然约束政策效果。

`60+ nondigital unassisted`在C1和C2中无法获得券；C3虽提供有限社区渠道，但3张老年保留券仍需在可触达老人之间竞争，本轮10 seeds中该组没有形成稳定核销结果。

## 口径修复

实验过程中修复了公交道路流量汇总：`scheduled_bus_vehicle_trips`现在来自完整工作日时刻表，不再只统计恰好有Agent出发的时间箱。因此政策不会通过乘客选择错误地改变公交班次数，道路车辆量只会因成功网约车车辆订单等真实车辆变化而改变。

## 输出

- `system_per_seed.csv`：每seed政策宏观结果。
- `system_distributions.csv`：均值、标准差、中位数、最小值和最大值。
- `policy_changes_vs_c0.csv`：各政策相对C0的变化。
- `group_per_seed.csv`、`group_distributions.csv`：年龄与老年数字接入组结果。
- `coupon_allocations.csv`、`coupon_outcomes.csv`：触达、参与、获券、绑定、核销及失效。
- `ride_hailing_dispatch.csv`：派单和车辆审计。
- `consistency_checks.csv`：券池、共同输入、共同优先值和车辆守恒检查。
