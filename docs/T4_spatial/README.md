# T4：九区合成功能城市与 home-zone 安置

## 模型定位

该九区结构并非上海行政区划的缩略图，而是参考上海的“西部传统中心—东部综合副中心—外围新城—产业园区—远郊弱势区”功能结构构建的合成城市，用于政策和网约车出行实验。

九区目标面积合计 `2587.698 km²`，约为上海市域面积的一部分。面积、边界、质心、人口权重和年龄结构均为可审计的合成参数，不代表真实行政区或真实通勤矩阵。

|---|---|---|---:|---:|

| Z2 | 北部内城混合区 | Z1 北侧/西北侧 | 148 | 238 km² |
| Z3 | 南部内城老城区 | Z1 南侧/西南侧 | 132 | 220 km² |
| Z4 | 东北外围居住区 | 东北外围 | 111 | 254 km² |
| Z5 | 东南外围居住区 | 东南外围 | 132 | 288 km² |
| Z6 | 西南产业新城 | 西南外围产业就业节点 | 129 | 304 km² |
| Z7 | 东部综合副中心 | Z1 东侧 | 174 | 538 km² |
| Z8 | 西北近郊老龄居住区 | 西北近郊 | 68 | 294 km² |

Z6 与 Z3、Z1 有明确交通联系，并通过唯一外联道路连接 Z9。Z9 标记为 `is_detached=true`，其跨区有效距离使用 `network_distance_multiplier=1.35`，表达道路绕行和交通弱势，而不是把欧氏直线距离等同于可达距离。

## 配置与派生量

所有假设位于 `config/shanghai_synthetic_city.json`：

- `target_area`：显式区域目标面积；
- `radial_distance_from_center` 与 `angular_position_degrees`：质心相对位置；
- `zone_type` 与 `display_name`：功能定义；
- `connected_to`：概念交通邻接；
- `network_distance_multiplier`：道路网络绕行系数；
- `residential_density_factor`：相对居住容量，不是人口密度实测值；
- `base_age_composition`：合成年龄结构。

年龄结构同时表达住房成本、就业集聚和人口沉淀机制。Z1、Z7 是年轻就业人口集聚区；Z6 既是产业就业节点，也容纳较多年轻劳动人口。Z4、Z5 保留外围居住区中为降低住房成本而跨区通勤的年轻人口。Z3 的老城区人口沉淀、Z8 的近郊老龄居住特征和 Z9 的年轻人口外流，使这三个区域的老年人口比例高于全市。配置中的分区比例是合成机制参数，随后会统一校准，使全市仍严格满足 18–39 岁 40%、40–59 岁 33%、60+ 27% 的总体目标。

质心坐标仍由极坐标全精度计算：

```text
x = r * cos(theta)
y = r * sin(theta)
```

区域面积直接读取 `target_area`，不再由不完整的圈层覆盖率推算。等效半径和区内平均距离仅用于 T6 的基线距离近似，不代表区域是圆形。

## 人口与年龄配额

```text
residential_population_capacity = target_area * residential_density_factor
population_weight = capacity / sum(capacity)
```

`allocate_zone_age_quotas()` 使用二维受控取整，同时保证九区人口总和、三年龄组总和与总 Agent 数严格一致。1000 Agent 配置的九区总数为：

```text
Z1 55, Z2 148, Z3 132, Z4 111, Z5 132,
Z6 129, Z7 174, Z8 68, Z9 51
```

home-zone 分配继续使用稳定 SHA-256 随机源；固定 seed 时，输入顺序不会改变个人安置结果。

## 审计边界

`build_spatial_audit()` 报告面积、人口权重、质心欧氏距离、区内距离和整数配额。T6 在欧氏距离基础上应用道路网络系数。当前模块仍不生成真实多边形边界、路网最短路径、正式 OD、交通方式或拥堵。
