# 200 Agent结果

## 实验用途

200人实验沿用50人的Agent比例和行为规则，不增加地铁，也不改变S1/S2简化空间。需求扩大后，使用按规模同比例扩大的车辆基线：工作日S1=32、S2=20，休息日S1=28、S2=16。该供给只用于机制比较，不是上海车辆估计。

实验重点观察以下涌现链条：

```text
优惠或派单规则
→ 个体方式选择与网约车请求变化
→ 有限车辆竞争、等待、失败和fallback
→ 公交与网约车共同改变道路压力
→ 活动完成、群体分配和热风险变化
```

## 已比较政策

### 优惠券 C0–C3

- `C0_no_coupon`：无券基线；
- `C1_public_limited`：全年龄有限公共券；
- `C2_elder_limited`：老年定向券；
- `C3_mixed`：公共券与老年保留券混合。

主实验使用40张八折券和30 seeds。公共券增加请求与道路压力；老年定向券的系统需求增量较小；混合券处于两者之间。

### 公共券覆盖扫描

固定Agent、活动、天气、车辆池和共同随机数，只改变公共券池为0、20、40、60、80张，对应约0%–40%的每日覆盖。3-seed扫描显示中等覆盖后等待和失败可能重新上升，但这里只能称为边际转折候选，不是最优覆盖率。

### 派单优先 R0–R2

- `R0_first_come`：按实际请求时间优先；
- `R1_elder_medical_priority`：同一请求时间下老年医疗优先；
- `R2_all_elder_priority`：同一请求时间下全部老人优先。

R1在本次样本中几乎没有改变结果，说明过窄规则可能无法触发。R2改善可叫车老人的等待和成功率，同时把部分失败转移给18–59岁乘客，体现有限运力下的资源再分配。后到请求不能抢占先到请求。

## 文件

- `policy_summary.csv`：C0–C3，200人、30-seed平均结果；
- `coupon_coverage_summary.csv`：0%–40%公共券覆盖扫描；
- `dispatch_priority_summary.csv`：R0–R2分年龄与数字接入组结果；
- `experiment_metadata.json`：可复现实验的主要参数。

这些结果验证机制是否存在，不应解释为现实人口阈值、平台最优派单规则或上海政策预测。向完整模型迁移时，应重新校准人口、OD、公交班次、车辆池、道路容量、活动需求与天气参数。
