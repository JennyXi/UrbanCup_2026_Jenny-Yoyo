# 200-Agent网约车供给比例与正式规模迁移

## 当前机制基线

200-Agent三seed筛选得到的暂定主供给为36辆，等于0.18辆有效初始车辆/Agent，或每100 Agents 18辆。24辆即0.12辆/Agent，保留为紧供给敏感性。

这里的车辆是九区模拟日开始时可进入时空守恒调度的有效车辆池，不是居民持有量，也不是现实上海每万人网约车数量。车辆可在一天内服务多单并停留在订单目的地区域。

## 正式规模的第一步换算

正式Agent数量确定后，可先用以下机制比例生成起始值：

```text
provisional_vehicle_pool = round(0.18 * formal_agent_count)
```

这个数值仅是保证机制与200-Agent原型具有相近初始供给强度的起点，不能直接作为正式基线。

## 正式规模必须重新校准

正式运行还必须同时考虑：

- 每个Agent代表多少真实居民；
- 模型产生的网约车订单/居民·日；
- 模拟覆盖的小时数及高峰并发请求；
- 每辆车每天完成的订单数；
- 正式空间中各区需求和车辆初始分布；
- 平台是否实现跨区空驶接客或车辆重定位；
- 现实统计中的活跃车辆、同时在线车辆与合规车辆口径差异。

因此正式实验至少保留三档：现实数据锚定的中供给、较低供给和较高供给。报告政策结论时优先使用请求/车辆、每车订单、成功率、等待、必要活动完成率和道路V/C等可缩放指标，不把36辆解释为现实最优规模。
