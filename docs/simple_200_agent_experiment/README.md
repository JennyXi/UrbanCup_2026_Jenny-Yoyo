# 200 Agent 简化机制实验

## 定位

这是一套两区域、三方式的机制验证环境，用来确认 Agent 选择、有限交通资源与政策之间能否形成可解释的宏观反馈。它不是完整上海模型，也不用于现实预测或政策参数校准。后续完整模型可以复用已经验证的机制，但需要恢复更完整的空间、交通网络和实证参数。

## 最小结构

- 空间：S1、S2；
- 方式：步行、公交、网约车；
- 天气：W0正常、W1高温、W2强降雨；
- 日期：工作日、休息日；
- Agent：200人，具有年龄、就业、数字接入、家庭协助和活动需求差异；
- 交通反馈：有限车辆守恒、S1/S2车辆移动、公交与网约车共享道路、一次fallback；
- 活动结果：完成、天气取消、交通未满足；
- 暴露结果：户外分钟、热剂量和年龄加权热风险。

## 已验证的机制

1. 恶劣天气降低步行并增加机动化需求，W2车辆竞争最明显。
2. 有限网约车在S1/S2间随订单移动，形成区域车辆失衡、等待和失败。
3. 公交和fallback吸收部分网约车失败，保护多数必要活动。
4. 公共优惠券诱导新增网约车需求，并稳定增加道路车辆量和拥堵压力。
5. 简化覆盖率扫描在20%–30%之间出现边际通行时间反转候选，只用于展示机制。
6. 社区代叫车允许少量非数字无协助老人进入叫车系统，但不改变其数字身份。
7. 老年优先派单改善可叫车老人的等待与成功率，同时把部分失败转移给其他乘客。

## 精简结果文件

运行：

```text
python -B -X utf8 -m scripts.build_simple_200_agent_summary
```

生成：

- `policy_summary.csv`：C0–C3的200人、30-seed总体结果；
- `coupon_coverage_summary.csv`：0%–40%公共券覆盖率的3-seed机制扫描；
- `dispatch_priority_summary.csv`：R0–R2按年龄与数字接入组的3-seed结果。

完整逐seed和逐请求审计仍保留在原输出目录，但简化实验汇报只需使用以上三张表。

## 使用边界

- 40%社区渠道是机制情景，不是上海实测覆盖率；
- 优惠券覆盖率反转不是现实最优值；
- 优先派单只重排同一实际请求时间的订单；
- 没有metro、九区域、司机上下线、主动重定位或跨天学习；
- 完整模型需要重新校准人口权重、活动、OD、运力、天气和政策参数。

## 向完整模型迁移

优先迁移有限车辆守恒、道路共享反馈、fallback、数字接入约束和政策审计字段。两区域距离、车辆数、券池规模和行为偏好只属于本简化实验，不应直接复制为完整模型的正式参数。
