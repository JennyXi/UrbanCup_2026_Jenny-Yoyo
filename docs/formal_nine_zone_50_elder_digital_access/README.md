# 九区50-Agent老年数字接入实验

本实验在12辆网约车候选基线下比较D0–D4老年数字接入和代叫机制。它使用W0、W2工作日与seeds 47–56，是机制诊断，不是上海数字政策效果预测。

## 政策

- `D0_baseline`：保持T1生成的数字接入和家庭协助。
- `D1_targeted_digital_training_75pct`：仅在已有智能手机的老人中培训，使数字接入达到75%，优先基线无协助老人。
- `D2_family_assistance_90pct`：家庭代叫覆盖提高到约90%。
- `D3_universal_elder_digital_access`：必要时提供设备，使老年数字接入达到100%。
- `D4_limited_community_phone_25pct`：仅对基线非数字且无家庭协助老人提供25%的社区/电话代叫概率，不改变其数字身份。

政策只改变网约车是否可选。它们不改变费用、天气偏好、年龄偏好、派单优先级、车辆数和天气取消概率，也不会强制老人打车。

## 接入结果

每个seed固定13位老人：

| 指标（每seed均值） | D0 | D1 | D2 | D3 | D4 |
|---|---:|---:|---:|---:|---:|
| 数字接入老人 | 6.0 | 10.0 | 6.0 | 13.0 | 6.0 |
| 有家庭协助老人 | 8.5 | 8.5 | 12.0 | 8.5 | 8.5 |
| 新增数字接入 | 0 | 4.0 | 0 | 7.0 | 0 |
| 新增家庭协助 | 0 | 0 | 3.5 | 0 | 0 |
| 社区/电话代理 | 0 | 0 | 0 | 0 | 0.1 |

基线非数字无协助组在9/10个seeds中存在，出现时平均约2.33人。D4采用独立25%覆盖抽样，在本组小样本中仅产生1个实际代理记录，因此不能据此评估社区渠道的真实效果。

## 交通结果

尽管接入属性明显改变，D1–D4在W0和W2下的交通结果均与D0相同。W2所有政策均为：

- 网约车请求13.0次；
- 成功11.6次、失败/fallback 1.4次；
- 每请求平均等待11.97分钟；
- 网约车方式份额12.39%；
- 道路车辆量1030.54；
- 平均总通行时间64.22分钟；
- 必要活动完成率97.05%。

老年组内部，基线数字老人W2平均只有0.1次网约车请求；非数字有协助和非数字无协助组均为0。D1和D3使部分基线无接入老人获得独立数字入口，但他们仍选择公交或地铁，没有产生新增网约车请求。

## 解释

该结果不表示“数字接入政策在现实中无效”。它说明在当前50-Agent简化模型中：

1. 家庭协助已经覆盖平均8.5/13位老人，数字接入与现有代叫能力部分重叠；
2. 老年人的公交偏好、网约车价格和轨道/公交可达性比“是否能打开平台”更能决定方式选择；
3. 基线老年必要活动在有观测的seeds中已全部完成，没有可供数字政策改善的未满足风险；
4. 非数字无协助组和D4实际覆盖人数很小，不能支持强政策排序。

因此，本实验成功识别出当前模型的边界：只扩大技术入口不足以产生宏观交通变化。下一轮若要评估数字政策的必要活动价值，应使用独立的老年必要出行压力情景，给各接入组相同的medical/必要活动需求，并保持正式主日历不变。
